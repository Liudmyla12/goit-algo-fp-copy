# goit-algo-fp

Фінальний проєкт курсу **«Алгоритми та структури даних» (GoIT)**.  
Репозиторій містить **7 завдань**, що охоплюють: зв’язні списки, рекурсію (фрактал), графи (алгоритм Дейкстри), бінарні купи (візуалізація), обходи дерева (DFS/BFS без рекурсії), greedy vs DP та метод Монте-Карло (дві кості).

## Структура репозиторію

```text
goit-algo-fp/
├── task_1.py
├── task_2.py
├── task_3.py
├── task_4.py
├── task_5.py
├── task_6.py
├── task_7.py
├── requirements.txt
├── .gitignore
├── pythagoras_tree.png
├── heap.png
├── dice_probabilities.png
└── traversal_output/
    ├── dfs_01.png ... dfs_06.png
    └── bfs_01.png ... bfs_06.png


⸻

Вимоги
	•	Python 3.10+
	•	Залежності з requirements.txt:
	•	numpy
	•	matplotlib
	•	networkx

⸻

Встановлення

Windows (PowerShell)

py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt

macOS / Linux

python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt

Порада: запускай скрипти в активованому venv, щоб не отримати ModuleNotFoundError.

⸻

Швидкий запуск

py .\task_1.py
py .\task_2.py --depth 10
py .\task_3.py
py .\task_4.py --show
py .\task_5.py
py .\task_6.py --budget 100
py .\task_7.py --n 200000 --plot


⸻

Опис завдань

Task 1 — Однозв’язний список (reverse / sort / merge)

Файл: task_1.py

Реалізовано:
	•	Reverse (in-place): реверс через зміну next-посилань між вузлами (без створення нового списку)
	•	Sort: сортування однозв’язного списку алгоритмом merge sort
	•	Merge: злиття двох відсортованих списків в один відсортований

Запуск:

py .\task_1.py


⸻

Task 2 — Фрактал “дерево Піфагора” (рекурсія)

Файл: task_2.py
Артефакт: pythagoras_tree.png

Глибина рекурсії задається аргументом --depth.

Запуск:

py .\task_2.py --depth 10


⸻

Task 3 — Алгоритм Дейкстри з бінарною купою (heapq)

Файл: task_3.py

Пошук найкоротших шляхів у зваженому графі:
	•	алгоритм Дейкстри
	•	оптимізація вибору наступної вершини через heapq (бінарна купа)

Запуск:

py .\task_3.py


⸻

Task 4 — Візуалізація бінарної купи (heap → tree)

Файл: task_4.py
Артефакт: heap.png

З масиву купи будується бінарне дерево за індексами:
	•	left = 2*i + 1
	•	right = 2*i + 2

Параметр --show відкриває вікно (і паралельно зберігає файл).

Запуск:

py .\task_4.py --show


⸻

Task 5 — Візуалізація обходів DFS/BFS (без рекурсії)

Файл: task_5.py
Артефакти: traversal_output/dfs_*.png, traversal_output/bfs_*.png

Реалізовано:
	•	DFS через стек
	•	BFS через чергу

Кожен крок обходу зберігається як окремий кадр. Колір вузлів змінюється від темного до світлого (HEX RGB) згідно порядку відвідування.

Запуск:

py .\task_5.py


⸻

Task 6 — Жадібний алгоритм vs Динамічне програмування (вибір їжі)

Файл: task_6.py

Два підходи:
	•	greedy_algorithm — вибір за максимальним співвідношенням calories / cost
	•	dynamic_programming — оптимальний набір у межах бюджету (DP)

Запуск:

py .\task_6.py --budget 100


⸻

Task 7 — Монте-Карло: два кубики + порівняння з аналітикою

Файл: task_7.py
Артефакт: dice_probabilities.png

Виконується:
	•	симуляція N кидків двох кубиків
	•	оцінка ймовірностей сум 2..12
	•	порівняння з аналітичними значеннями (таблично)
	•	побудова графіка при --plot

Запуск:

py .\task_7.py --n 200000 --plot


⸻

Артефакти
	•	pythagoras_tree.png (Task 2)
	•	heap.png (Task 4)
	•	dice_probabilities.png (Task 7)
	•	traversal_output/ — покрокові кадри DFS/BFS (Task 5)

⸻

Висновки
	•	Усі 7 завдань реалізовано та запускаються окремими скриптами.
	•	Проєкт відтворюється в “чистому” середовищі після встановлення залежностей з requirements.txt.
	•	Візуалізаційні завдання генерують PNG-артефакти, додані в репозиторій як підтвердження виконання.
	•	Для методу Монте-Карло похибка в середньому зменшується зі зростанням кількості симуляцій приблизно як 1 / √N.


```
